# ASIN 排名追踪器 - 产品需求文档 (PRD)

**版本**: 2.0 (Python 技术栈版)  
**创建时间**: 2026-04-12  
**技术栈**: Vue.js + Element UI / Python FastAPI + Playwright / MySQL

---

## 一、产品概述

### 1.1 产品定位

ASIN 排名追踪器是一款面向亚马逊卖家的 SEO 分析工具，帮助用户追踪其产品在关键词搜索结果中的自然排名和广告排名，为优化产品 listing 和广告投放提供数据支持。

### 1.2 核心价值

- **自动化排名追踪**: 批量关键词自动搜索，无需手动查询
- **双维度排名**: 同时追踪自然排名和广告排名
- **可视化展示**: 表格形式清晰展示排名数据
- **高效并发**: 基于 asyncio + Semaphore 实现高效并发爬取

### 1.3 目标用户

- 亚马逊第三方卖家
- 电商运营人员
- 跨境电商营销人员
- 产品 SEO 优化师

## 二、功能需求

### 2.1 核心功能

#### F1: 关键词批量输入
- **描述**: 用户可以在文本域中输入多个关键词，每行一个
- **输入**: 多行文本，每行一个关键词
- **约束**: 
  - 最少 1 个关键词
  - 最多 100 个关键词
  - 每个关键词长度 1-200 字符
- **优先级**: P0

#### F2: ASIN 输入与验证
- **描述**: 用户输入要追踪的 ASIN
- **输入**: ASIN 字符串（10 位字母数字组合）
- **验证**: 
  - 格式验证（10 位，字母数字）
  - 自动转换为大写
- **优先级**: P0

#### F3: 翻页数配置
- **描述**: 用户配置最大搜索翻页数
- **输入**: 数字选择器
- **范围**: 1-50 页，默认 5 页
- **优先级**: P0

#### F4: 亚马逊搜索爬取
- **描述**: 系统自动对每个关键词执行亚马逊搜索
- **处理**:
  - 使用 Playwright 模拟浏览器
  - 支持多站点（amazon.com, amazon.co.uk 等）
  - 遵守请求频率限制
- **优先级**: P0

#### F5: 排名数据提取
- **描述**: 从搜索结果中提取 ASIN 的排名信息
- **输出**:
  - 自然排名：页码 + 页内位置
  - 广告排名：页码 + 页内位置
  - 状态：找到/未找到
- **优先级**: P0

#### F6: 结果表格展示
- **描述**: 以表格形式展示所有关键词的排名结果
- **列**: 关键词、自然排名、广告排名、状态、操作时间
- **功能**: 排序、筛选、导出
- **优先级**: P0

#### F7: 任务管理
- **描述**: 支持任务的创建、查看、取消
- **功能**:
  - 任务列表
  - 任务进度实时显示
  - 任务历史查询
- **优先级**: P1

### 2.2 辅助功能

#### F8: 数据导出
- **描述**: 支持导出排名数据为 CSV/Excel
- **格式**: CSV, XLSX
- **优先级**: P1

#### F9: 定时任务
- **描述**: 支持设置定时自动追踪
- **功能**: 每日/每周自动执行
- **优先级**: P2

## 四、输入输出定义

### 4.1 输入参数

| 参数 | 类型 | 必填 | 说明 | 约束 |
|------|------|------|------|------|
| asin | string | 是 | 产品 ASIN | 10 位字母数字，自动转大写 |
| keywords | string[] | 是 | 关键词列表 | 1-100 个，每行一个 |
| maxPages | number | 否 | 最大翻页数 | 1-50，默认 5 |
| site | string | 否 | 亚马逊站点 | amazon.com/amazon.co.uk 等，默认 amazon.com |

### 4.2 输出数据

#### 任务提交响应
```json
{
  "taskId": "uuid-string",
  "status": "pending",
  "createdAt": "2026-04-12T10:00:00Z",
  "totalKeywords": 10,
  "maxPages": 5
}
```

#### 排名结果
```json
{
  "keyword": "wireless earbuds",
  "organicPage": 3,
  "organicPosition": 12,
  "adPage": 1,
  "adPosition": 4,
  "status": "found",
  "timestamp": "2026-04-12T10:05:00Z"
}
```

#### 状态枚举
- `pending`: 等待执行
- `running`: 执行中
- `completed`: 已完成
- `failed`: 失败
- `cancelled`: 已取消

## 五、边界条件

### 5.1 输入边界

| 场景 | 处理方式 |
|------|----------|
| 关键词为空 | 返回错误提示"请至少输入一个关键词" |
| 关键词超过 100 个 | 截断前 100 个，提示用户 |
| ASIN 格式错误 | 返回错误提示"ASIN 格式不正确" |
| 翻页数小于 1 | 自动修正为 1 |
| 翻页数大于 50 | 自动修正为 50 |

### 5.2 爬取边界

| 场景 | 处理方式 |
|------|----------|
| 亚马逊返回验证码 | 暂停任务，记录错误，继续下一个关键词 |
| 网络超时 | 重试 3 次，失败后标记为 failed |
| 页面结构变化 | 记录错误日志，继续执行 |
| 请求频率过高 | 自动增加延迟，遵守 robots.txt |

### 5.3 性能边界

| 指标 | 目标值 |
|------|--------|
| 单关键词处理时间 | < 30 秒（5 页内） |
| 并发任务数 | 最多 5 个并行任务 |
| 请求延迟 | 2-5 秒随机延迟 |
| 数据库查询响应 | < 100ms |

## 六、非功能需求

### 6.1 性能需求

- 支持每秒处理 2-3 个关键词（考虑反爬延迟）
- 前端页面加载时间 < 2 秒
- API 响应时间 < 200ms（不含爬取）

### 6.2 安全需求

- API 限流：每 IP 每分钟最多 60 次请求
- 输入验证：所有输入参数进行严格验证
- 数据加密：敏感数据（如代理配置）加密存储
- XSS 防护：前端输入输出进行转义

### 6.3 可用性需求

- 系统可用性 > 99%
- 支持断点续跑（任务中断后可恢复）
- 提供清晰的任务进度反馈

### 6.4 可靠性需求

- 任务失败自动重试（最多 3 次）
- 数据库操作事务保证
- 异常日志完整记录

### 6.5 可维护性需求

- 代码注释覆盖率 > 60%
- 模块化设计，便于扩展
- 完善的日志系统

### 6.6 可扩展性需求

- 支持添加新的亚马逊站点
- 支持添加新的电商平台（eBay、Walmart 等）
- 支持分布式部署

## 七、合规要求

### 7.1 亚马逊 robots.txt

- 遵守亚马逊 robots.txt 规定
- 不在禁止爬取的路径上请求
- 尊重 Crawl-delay 指令

### 7.2 请求频率控制

- 单 IP 请求间隔 2-5 秒
- 支持代理池轮换
- 支持 User-Agent 轮换

### 7.3 数据使用

- 爬取数据仅用于个人分析
- 不存储亚马逊页面完整 HTML
- 不将数据用于商业转售
